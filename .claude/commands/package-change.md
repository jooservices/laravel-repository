# package-change

Use `AGENTS.md` and the relevant `.github/skills/` files before implementing a non-trivial package or repository-governance change.

Focus on minimal, module-appropriate changes and keep tests, docs, CI references, and AI adapters aligned.

For request-query or repository lifecycle work, preserve strict allowlists, max per-page guards, criteria behavior, query reset semantics, and the low-level cache wrapper boundary.
